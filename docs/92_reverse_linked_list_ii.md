# 92. 反转链表 II (Reverse Linked List II)

> **📅 日期**：2026-03-11
> **💡 难度**：🟡中等
> **🏷️ 考点**：链表、指针操作、原地反转

---

## 1. 题目介绍
* **题目链接**：[LeetCode 92. Reverse Linked List II](https://leetcode.cn/problems/reverse-linked-list-ii/)
* **核心目标**：给你单链表的头指针 `head` 和两个整数 `left` 和 `right` ，其中 `left <= right` 。请你反转从位置 `left` 到位置 `right` 的链表节点，返回反转后的链表

---

## 2. 解题思路

### 🟢 我的直观思路
* **想法**：先遍历一遍链表，找到反转区间的开始和结束，将其断开，单独反转后再进行两次拼接
* **缺点**：逻辑过于复杂，需要多次处理断链后的头尾连接，反复进行遍历，稍不注意就会丢掉节点导致内存泄漏或空指针，思维必须一直清晰，且无法达到指定空间复杂度要求

### 🚀 优化思路
* **核心逻辑**：不移动节点，只修改指针指向。通过三个关键指针 `pre`, `cur`, `next` 在一次遍历中完成
* **关键步骤**：
    1. 使用 **dummy**处理 `left=1`（头节点变动）的情况
    2. **定位 pre**将 `pre` 移动到待反转区间的前一个位置
    3. **循环换位**：在反转区间内，不断将 `cur` 后的节点（`next`）放到 `pre` 之后
* **复杂度分析**：
    * **时间复杂度**：$O(n)$，只需遍历一次
    * **空间复杂度**：$O(1)$

### 💎 有何收获
* **dummy的引入**：有了 dummy，原链表的 head 就变成了 dummy 的 next,原链表的 head 也有了前驱节点
* **存后继**：改向必先存址，不能将链表丢失
* **防御性编程**：在写每一行 -> 时考虑前驱和后继是否为空；注意赋值与关系运算符
* **链表是灵活的**：更深刻理解了链表不同于数组的灵活性，对指针的操作也有了更好的理解
---

## 3. 我的错误总结

> **总结：** 忽略了链表逻辑上的一种连续性，不需要进行大量元素的挪动即可实现反转等操作。要注意的是时刻找得到链表，不能让它洒落一地。

| 错误点 | 原因分析 | 解决方案 |
| :--- | :--- | :--- |
| **断链/节点丢失** |循环中修改指针顺序错误，导致后面一段链表断开找不到地址| 谨记：一抓(`next`)、二跨(`cur->next`)、三插(`next->next`)、四接(`pre->next`)|
| **将第三步写为`next->next = cur`** |仅仅注意到了第一次循环的情况，没有考虑到后续循环|涉及链表指针操作时要动手画一画|
| **头节点返回错误** |`left=1` 时 `head` 已经不是原头节点了|返回 `dummy.next` 而非 `head`|
