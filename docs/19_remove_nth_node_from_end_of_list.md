# 19. 删除链表的倒数第 N 个结点 (Remove Nth Node From End of List)

> **📅 日期**：2026-03-16
> **💡 难度**：🟡 中等
> **🏷️ 考点**：双指针、快慢指针

---

## 1. 题目介绍
* **题目链接**：[LeetCode 19. Remove Nth Node From End of List](https://leetcode.cn/problems/remove-nth-node-from-end-of-list/)
* **核心目标**：给你一个链表，删除链表的倒数第 `n` 个结点，并且返回链表的头结点，尝试使用一趟扫描实现

---

## 2. 解题思路

### 🚀 思路：快慢指针
* **核心逻辑**：利用距离差在一次遍历中定位
* **关键步骤**：
    1. **引入 Dummy**：处理头节点
    2. **先行**：`fast` 先走 $n+1$ 步
    3. **并进**：`fast` 和 `slow` 同时走，直到 `fast` 为 `NULL`。
    4. **删除**：此时 `slow` 位于目标节点前驱，执行 `slow->next = slow->next->next`。

### 📊 指针

| 指针 | 初始位置 | 最终位置 | 职责 |
| :--- | :--- | :--- | :--- |
| **`dummy`** | 虚拟头节点 | 虚拟头节点 | 保护原头节点，防止删除头节点时断链。 |
| **`fast`** | `dummy` | `NULL` | 通过先跑 $n$ 步来制造距离。 |
| **`slow`** | `dummy` | 待删节点的前驱 | 精准定位并执行删除动作。 |

---

## 3. 我的错误总结

> **总结：** `fast` 多走一步，`slow` 就能少走一步，从而停在“前驱”而非“目标”上。

| 错误点 | 原因分析 | 解决方案 |
| :--- | :--- | :--- |
| **空指针异常** | 未使用 dummy 时，删除头节点会导致 `head` 丢失 | 养成使用 **dummy** 的习惯 |
| **定位偏移** | `fast` 走的步数不够，导致 `slow` 停在目标节点本身 | 确保 `slow` 停在目标前一个 |
